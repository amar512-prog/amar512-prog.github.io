Reasoning with Sampling: why better cuts matter more than more tokens

This paper makes a simple but important claim: not every token in a reasoning trace matters equally. If you want better inference-time reasoning, spend your resampling budget at the moments where the model is choosing a strategy, not where it is just rewording a sentence.

Imagine you are solving this coding problem:

Given an array, find the longest subarray with sum less than or equal to K.

At the start, you have multiple possible strategies:

  1. Brute force.
  2. Prefix sum.
  3. Sliding window.
  4. Binary search over the answer.

The most important decision is not whether you write variable `i` or `j`. The real decision is choosing the algorithmic strategy. That is the intuition behind Reasoning with Sampling: Cutting at Decision Points.

A Simple Example First

Suppose an LLM is solving this problem:

Find the sum of the first `n` natural numbers.

The model might reason like this:

We can solve this using a loop. Initialize `sum = 0`. For `i` from `1` to `n`, add `i` to `sum`. Return `sum`.

A stronger reasoning trace would be:

Use the formula for arithmetic series. The sum of the first `n` natural numbers is `n(n + 1) / 2`. Return that formula directly.

Both traces are correct, but the second one reflects a better strategic choice. The key decision point happened early, when the model chose the method itself.

What Problem Is This Paper Solving?

The paper studies whether we can improve reasoning quality at inference time without retraining the model. Earlier sampling methods already showed that multiple reasoning traces can help, but many of them resample from random token positions. That is inefficient because most tokens are not strategic choices.

What Is Reasoning with Sampling?

Reasoning with sampling means generating a reasoning trace, revisiting part of it, and resampling the continuation instead of changing model weights.

  1. Generate a reasoning trace.
  2. Choose a cut point.
  3. Resample the suffix.
  4. Accept or reject the new trace.
  5. Repeat.

What Is a Reasoning Trace?

We need to find the maximum profit. First track the minimum price so far. Then calculate current profit for each day. Update the best answer.

A reasoning trace is the sequence of tokens the model emits while solving a problem. Some parts contain real choices. Others are just local detail.

What Is a Decision Point?

A decision point is where the model chooses a major path.

  • Use induction versus use contradiction.
  • Use dynamic programming versus use two pointers.
  • Split into cases versus apply Bayes' theorem.

Changing one of these choices can change the entire answer. Changing a random late token usually just changes wording.

Why Is Random Cutting Weak?

We should use dynamic programming. Let `dp[i]` represent the best answer up to index `i`. Now we iterate from left to right. Finally return `dp[n]`.

If you cut at the line about iterating left to right, you mostly rewrite implementation detail. If you cut at the first line, you give the model a chance to try an entirely different strategy.

What Is Entropy?

Entropy is a measure of uncertainty. If the model is very confident about the next token, entropy is low. If several continuations look plausible, entropy is high.

That is useful because major reasoning choices often happen exactly where the model is uncertain among several strategies.

What Is an Entropy Jump?

An entropy jump means uncertainty suddenly increases at a specific token position. The paper treats those spikes as a proxy for decision points.

What Is Entropy-Cut Metropolis-Hastings?

  1. Generate a reasoning trace.
  2. Compute entropy at each token position.
  3. Find positions with high entropy or sharp jumps.
  4. Cut there and resample the suffix.
  5. Use Metropolis-Hastings to accept or reject the proposal.
  6. Repeat.

The result is a sampler that spends more effort exploring meaningful strategy changes instead of cosmetic rewrites.

What Is Metropolis-Hastings in Simple Terms?

It is a sampling procedure that repeatedly proposes a new sample and decides whether to keep it. Here, the "sample" is an entire reasoning trace.

What Is the Power Distribution?

A power distribution is a sharpened version of the model's original probability distribution. High-probability traces become even more favored.

Answer Probability
Correct reasoning 0.50
Weak reasoning 0.30
Wrong reasoning 0.20

Why Does Cutting at Decision Points Help?

Long reasoning traces may contain only a handful of truly important decisions: which proof method to use, which equation to introduce, or which algorithmic structure to follow. Smarter cuts spend compute on those moments.

A Mathematical Toy Example

Suppose a 100-token reasoning trace contains only 5 true decision points. Uniform random cutting hits a useful spot with probability `5 / 100 = 0.05`. That means 95 percent of cuts are probably wasted.

Entropy-aware cutting changes the odds by deliberately searching near the places where the model is uncertain about strategy.

A Coding Example

Find whether an array contains two numbers whose sum equals a target.

A weak trace says:

Check all pairs using two nested loops. This gives `O(n^2)`.

A stronger trace says:

Use a hash set. For each number `x`, check whether `target - x` already exists. This gives `O(n)`.

The key decision is the algorithm choice itself, not the later phrasing of the steps.

How Is This Different from Reinforcement Learning?

RL post-training changes model weights through additional training. Entropy-Cut sampling keeps the weights fixed and improves reasoning during inference.

Method Changes model weights? Needs training? Happens during inference?
RL post-training Yes Yes No
Normal sampling No No Yes
Entropy-Cut sampling No No Yes

How Is This Different from Chain-of-Thought Prompting?

Chain-of-thought only asks the model to reason step by step. This paper goes further: generate a trace, inspect uncertainty, cut at important positions, and resample from there.

Where Could This Be Useful?

  • Competitive programming
  • Math problem solving
  • Scientific reasoning
  • Agent planning
  • Tool-using systems

What Are the Limitations?

  • Entropy is only a proxy for important decisions.
  • Repeated sampling still increases inference cost.
  • The base model must already contain useful reasoning paths.
  • Factual short-answer tasks may benefit less.
  • The implementation is more complex than ordinary decoding.

FAQ

What problem does this paper solve?

It makes inference-time reasoning more efficient by revisiting strategic decision points instead of sampling blindly from random token positions.

Why is uniform random cutting inefficient?

Because most tokens are not major decisions. They are just wording or small implementation detail.

Why does high entropy matter?

High entropy often appears when several reasoning strategies are plausible, which is exactly when the model may be standing at a meaningful fork.

What is the one-line takeaway?

Do not randomly rewrite reasoning. Revisit the points where the strategy was chosen.

Summary

The paper's message is simple: reasoning traces are not flat. A long trace contains a few real junctions and many unimportant tokens. Entropy-Cut Metropolis-Hastings uses uncertainty to find those junctions, then resamples from there so inference-time compute goes toward strategy changes, not cosmetic edits.
