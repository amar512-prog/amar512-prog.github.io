API foundations and a sample RESTful API in C++

This page is a compact API reference that starts with core concepts and ends with a small Pistache-based C++ example you can use as a practical template.

What is an API?

Application Programming Interfaces, or APIs, provide a way for different pieces of code, or a human and code, to interact with one another. In the context of the web, the term API is usually used for web APIs, where a client sends requests and a server responds.

Types of API Calls

  • Safe: a safe request does not modify data on the server.
  • Idempotent: a request returns the same result no matter how many times it is called. It may change the server only on the first call.

Structure of an API Request

Every API request needs an endpoint and an action. In web APIs, the endpoint is usually a URL because it identifies the resource the server should interact with. If an endpoint starts with `/` instead of `http` or `www`, it usually means the base URL has been left out.

API actions, also known as verbs, specify the desired operation on the resource. The most common actions are `GET`, `POST`, `PUT`, and `DELETE`. These are often grouped as CRUD: Create, Read, Update, and Delete.

Choosing the right action matters. `POST` is commonly used to create new objects, `GET` to read existing information, `PUT` to modify existing objects, and `DELETE` to remove them. `PATCH`, `OPTIONS`, and `HEAD` are also used, but less frequently.

API Parameters

API parameters help create standard structure in an API. They let similar things be grouped together and make it easier to retrieve specific information.

Request Parameters

Example:

GET https://api.github.com/users/:{username}/repos

In this example, `username` is a request parameter. These parameters act as replaceable strings in the API endpoint and are common in web APIs. They can also be represented using curly braces instead of a colon.

Query Parameters

Query parameters act as filters or additional actions that can be applied to an endpoint. They are represented by a question mark in the URL and use a key-value structure. 

GET https://api.github.com/user/repos?type=member

This example shows repositories that the authenticated user is a member of.

Headers, Body, and Response

Every API request includes headers that provide background information about the client sending the request.

Many requests, especially `POST` and `PUT`, also include a body describing the resource attributes. This body can be form data, encoded form data, raw data, binary data, or even GraphQL data.

The API response is sent by the server after it processes the request. A response can include a body, cookies, and headers.

Types of APIs

SOAP, or Simple Object Access Protocol, is an XML-based messaging protocol used to exchange structured data. It has more overhead than RESTful APIs and was designed for stricter enterprise-style integrations.

REST, or Representational State Transfer, is a lightweight web architecture and communication style widely used for web services. It is generally simpler to develop and maintain than SOAP.

Sample RESTful API in C++ Using Pistache

This section walks through a simple Ubuntu setup for building a RESTful API in C++ with the Pistache library. The goal is a tiny API that accepts two numbers and returns their sum.

Installing GCC and Pistache

Install GCC:

sudo apt-get install g++

Install Pistache:

sudo add-apt-repository ppa:pistache+team/stable
sudo apt update
sudo apt install libpistache-dev

Creating the API

Create a file called api.cpp with the following code:

#include <pistache/endpoint.h>
#include <pistache/http.h>
#include <pistache/router.h>
#include "rapidjson/document.h"
#include <iostream>

using namespace Pistache;

void add(const Rest::Request& req, Http::ResponseWriter resp)
{
  rapidjson::Document doc;
  doc.Parse(req.body().c_str());
  int x = doc.HasMember("x") ? doc["x"].GetInt() : 0;
  int y = doc.HasMember("y") ? doc["y"].GetInt() : 0;
  int sum = x + y;
  resp.send(
    Http::Code::Ok,
    "The result of addition of " + std::to_string(x) +
    " and " + std::to_string(y) +
    " is " + std::to_string(sum) + "\n"
  );
}

int main(int argc, char* argv[])
{
  using namespace Rest;
  Router router;
  Port port(9900);
  Address addr(Ipv4::any(), port);
  std::shared_ptr<Http::Endpoint> endpoint =
    std::make_shared<Http::Endpoint>(addr);
  auto opts = Http::Endpoint::options().threads(1);
  endpoint->init(opts);

  Routes::Post(router, "/add", Routes::bind(&add));

  endpoint->setHandler(router.handler());
  endpoint->serve();
}

This code creates an endpoint listening on port `9900`. When a `POST` request is sent with a JSON payload containing `x` and `y`, the API returns the sum of the two numbers.

Compiling the Code

A simple Makefile for compiling the example:

# C++17 required for Pistache!
CCFLAGS=-O2 --std=c++17 -lpistache

all: server

server:
    g++ api.cpp $(CCFLAGS) -o server

clean:
    rm server

Running the API

./server

The API should now be running on localhost:9900.

Testing the API

You can test it with curl:

curl -i -X POST -H "Content-Type: application/json" \
  -d '{"x": 5, "y": 3}' \
  http://localhost:9900/add

This sends a JSON payload with `x = 5` and `y = 3` and should return the sum, which is `8`. It is also worth testing incorrect types or missing fields to make sure the API responds sensibly in edge cases.

This note works as a quick primer: understand endpoints, verbs, parameters, headers, and payloads first, then reinforce the ideas by building one tiny RESTful API yourself.
